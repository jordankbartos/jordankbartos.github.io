PokerCalc

I play a weekly game of $1-buy-in poker with some friends, but we rarely end the game with nice even amounts to transfer. So how to settle the debts with the fewest possible number of transfers? PokerCalc formulates the problem as a graph and solves it with a greedy algorithm.

Rewind Revitalize

Winner of the Oregon State University Hackathon Club's Winter 2018 BeaverHacks hackathon! Two other students and I created this program after our first two terms in the postbaccalaureate program. It is a C++ mindful journaling program with user accounts and encrypted user data (so saved data is not human-readable). Although we entered in a category for new students, we were astonished to find that we won first place overall!

ComplimentorBot

This is a silly uplifting slackbot I made because... well because I wanted to make a slackbot. Upon starting up, this slack-bot initializes its dictionary from three txt files that are included with the bot. Then, it loads the user handles of every user in the slack workspace. When it is issued the command, ComplimentorBot generates a random compliment targeted at a random workspace member or the person indicated.

Soil Moisture Sensor

I built this soil-moisture sensor as a side-project in the Summer following my first three quarters in Oregon State University's post-baccalaureate computer science program. It was one of the first hands-on projects I undertook in which I could see the actual results of my code (and my arduino build) in action afterwards. Plus I was always forgetting to water my house plants. This helped save my poor neglected house plants from a slow and thirsty demise.
